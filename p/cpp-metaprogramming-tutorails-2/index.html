C++ 模板编程无痛入门（二） Type Traits：类型特征核心实现与约定

Feb 12, 2023
3 minute read

在前一篇 C++ 模板编程无痛入门（一）前言、两种元函数 中，我们举了一个使用模板偏特化来优化 byte_swap 函数的代码困境。当时提到了我们需要能够识别传入数据的“类型本身的更多特征”来加强我们的模板。

这就是 Type Traits 。

在拥有了前文对于元函数的理解之后。本文将介绍一些特殊且重要的元函数，Type Traits 系统将建立在这些元函数之上。

一切的核心：std::integral_constant

这是 Type Traits 的核心元函数，因此需要特别记忆，有一丢丢复杂和抽象：

template <class T, T v> // 上篇文章已经提到过，这里可以优化为 auto
struct integral_constant {
    static constexpr T value = v;                     // 一个值元函数的返回值

    using value_type = T;                   
    using type       = integral_constant<T, v>;       // 一个类型元函数的返回值

    /// 这里定义了一个转换到T类型值隐式转换
    constexpr operator value_type() const noexcept {
        return value;
    }

    // 如果这个模板实例化了，可以使用括号操作符直接访问 value
    constexpr value_type operator()() const noexcept {
        return value;
    }
}

显然这既是一个值元函数，又是一个类型元函数。但目前看不出来这种设计有什么用，也不知道为何要定义隐式转换。让我们继续。

std::bool_constant、std::true_type、std::false_type

这三种元函数都是 std::integral_constant 的模板别名。std::bool_constant 别名了 std::integral_constant，而 std::true_type、std::false_type 是 std::bool_constant 的别名。

1
2
3
4
5
6

template <bool B>
using bool_constant = integral_constant<bool, B>;

using true_type = bool_constant<true>;

using false_type = bool_constant<false>;

其中，std::true_type、std::false_type 被称作零元元函数，因为没有传入参数，但是却能拿到返回值。

有了前一篇文章的介绍，相信各位读者已经能够在脑海中想象出这些元函数如何调用，以及用 ::type 和 ::value 分别能够拿到什么了。

那么进一步讨论这些元函数的用法，需要首先交代一些关于 type traits 的约定，这些约定一样来自于社区惯例，但现在已经成为标准：

一元类型特征（Cpp17UnaryTypeTrait）约定

  • utt.1 类模板
  • utt.2 只有一个类型模板参数（但可以添加一些附加参数）
  • utt.3 必须有默认构造函数
  • utt.4 必须可以复制构造
  • utt.5 必须 public 且没有歧义地继承 std::integral_constant
  • utt.6 同时，基类的成员需要保持没有歧义、可用，绝不可隐藏起来

二元类型特征（Cpp17BinaryTypeTrait）约定

二元类型特征与一元基本一致：

  • btt.1 类模板
  • btt.2 需有两个类型模板参数（但可以添加一些附加参数）
  • btt.3 必须有默认构造函数
  • btt.4 必须可以复制构造
  • btt.5 必须 public 且没有歧义地继承 std::integral_constant
  • btt.6 同时，基类的成员需要保持没有歧义、可用，绝不可隐藏起来

转换特征（Cpp17TransformationTrait）约定

转换特征则不同之处更多：

  • tt.1 类模板
  • tt.2 只有一个类型模板参数（但可以添加一些附加参数）
  • tt.3 需要有一个内嵌的、public的类型命名为type
  • tt.4 无需默认的构造函数，无需默认的拷贝构造函数
  • tt.5 没有继承要求

未定义的行为

这意味着一些事情，不要去做：

  • ub.1 不要特化标准库的类型特征，除了特殊声明可以特化的（如 common_type)
  • ub.2 需要当心不完整类型。可以用，但是必须保证示例化一个你的类型特征，无论这个类型是否是完整的，结果都一致。

不完整类型是这样一种类型，它缺乏足够的信息例如长度去描述一个完整的对象。如一个前向声明。

有了上面的约定。让我们开始介绍一些标准库类型特征系统中的重要元函数，并利用这些重要元函数的实现来理解这些约定。

一元类型特征：is_void

这是一个值元函数，判断一个类型是不是一个void类型。会产生 true_type 或者 false_type。

这个元函数是通过模板特化来实现的：

1
2
3
4
5
6
7
8

// 泛用情况
template <typename T>
struct is_void : std::false_type {}; // utt.5 

// 特化情况：
// 全特化模板，只针对void类型才会匹配到这里
template <>
struct is_void<void> : std::true_type {};  // utt.5

在之前文章的例子里面，我们通常都会使用一个 static_assert 断言来表示一个元函数的用法。那么 is_void 应该是如何使用的呢？is_void 继承自 std::true_type 或 std::false_type。所以编译器根据传入参数的类型（匹配泛用模板或特化模板）实际上是在使用后两者。

false_type原理一样。这里只以 true_type 为例。

我们已经知道了 true_type 仅仅是 integral_constant 的模板别名，它是一个 integral_constant<bool, true>，可以认为编译器生成的代码是长这样的：

struct integral_constant_true_type {
    static constexpr bool value = true;  // 编译期，这里已经“保存”了一个 true了                     

    // using value_type = bool;                   
    using type       = integral_constant<bool, true>;

    /// 这里定义了一个隐式转换转换成一个bool值
    constexpr operator bool() const noexcept {
        return value;
    }

    // 可以使用括号语法直接访问 value
    constexpr bool operator()() const noexcept {
        return value;
    }
}

因此，is_void 它的用法是这样的：

1
2

static_assert（is_void<void>{}); //使用花括号语法示例化了一个 is_void 对象（true_type)，又隐式转换成了一个 constexpr bool 值 true
static_assert（not is_void<int>{}); //使用花括号语法示例化了一个 is_void 对象（false_type)，又隐式转换成了一个 constexpr bool 值 false

template <typename T> struct is_void : std::false_type { };

template <> struct is_void<void const> : std::true_type { };
template <> struct is_void<void volatile> : std::true_type { };
template <> struct is_void<void const volatile>
: std::true_type {     };
//  标准库实现了简化写法
template <typename T>
inline constexpr bool is_void_v = is_void<T>::value;

转换类型特征 remove_const

字面意思，功能是 移除一个类型的const修饰词。

注意：只移除顶级const。这也就意味着如果你的指针是指向一个const类型的指针，但本身并不是const，那么remove_const将什么都不会做。

实现 remove_const，我们继承自上一篇文章中实现的 TypeIdentity：

1
2
3
4

template <typename T>
struct TypeIdentity {
    using type = T; 
}

同样需要利用特化：

1
2
3
4
5
6
7
8
9

// 泛用模式
template <typename T>
struct remove_const : TypeIdentity<T> { };
// 特化版本，这是一个偏特化，因为我们需要明显传入const关键词，剩下的部分 T 才是编译器推断的
template <typename T>
struct remove_const<T const> : TypeIdentity<T> { }; // 显然，丢掉了const
// 用于简化的模板别名 
template <typename T>
using remove_const_t = typename remove_const <T>::type;

转换类型特征 conditional

这个模板特征有三个参数，第一个参数是一个bool，第二个第三个参数都是一个类型。它的功能是如果第一个参数为true，返回第二个参数中的类型，反之返回第三个参数中的类型。

实现如下：

template <bool Condition, typename T, typename F>
struct conditional : TypeIdentity<T> { };
template <typename T, typename F>
struct conditional<false, T, F> : TypeIdentity<F> { };

static_assert(is_same_v<int,
                  conditional_t<is_void<void>::value, int, long>);
static_assert(is_same_v<long,
                  conditional_t<is_void<char>::value, int, long>);

小结

本文介绍了 Type Traits 类型特征的核心元函数 integral_constant 及其重要的模板别名。并介绍了三种类型特征的约定。并利用几个标准库特征模板：is_void、remove_const和 conditional来理解这些约定和类型特征的实现方法。
