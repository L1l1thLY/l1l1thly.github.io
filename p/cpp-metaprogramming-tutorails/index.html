本文是一个非常适合新手阅读的模板元编程的入门教程。但仍然需要你有一些基础的 C++ 能力，但不需要很多。

如果你像我一样是个C++菜鸡，但大概能够理解 C++ 的面对对象编程和范型的概念。那么阅读并理解本文内容仍然不成问题。

本文内容和代码例都来自cppcon的内容整理和个人理解总结，如有错误请一定拍砖，十分感谢。

模板元编程是什么？

这里有一个来自维基百科的定义：

模板元编程（英语：Template metaprogramming，缩写：TMP）是一种元编程技术，编译器使用模板产生暂时性的源码，然后再和剩下的源码混合并编译。这些模板的输出包括编译时期常量、数据结构以及完整的函数。

如果这个描述仍然让你感到费解，那么这里有另外一个定义：

  • 元编程是程序作者像正常程序操作数据一样操作程序本身的手段。

如果这句话仍然让你感到抽象。那你可以从性能的角度来理解：

  • 使用元编程能力，我可以把运行时需要做的事情，移动到编译期来做，提升我的程序性能——当然这反之会提升你的编译时间，正如我在文章最开头提到的那样。

或者，可以从你将要做的工作来理解：

  • 我想要编写一些给同事、同学或是社区使用的框架代码。这些代码我想尽可能简洁且灵活（最基础的范型提供了一些这样的能力，但深入使用模板元编程可以做得更多）。或者想给 C++ 本身添加一些能力，制作一些语法糖。

那也许，你将不得不了解一些元编程的技术。

模版特化与 Type Traits

简单了解过范型的就能知道，普通的函数只能提供给函数声明时指定的类型使用，而模板函数传入多种数据类型。

比如定义一个函数叫：

/// 以下代码示例来自 https://blog.aaronballman.com/2011/11/a-simple-introduction-to-type-traits/

template <typename T>
T byte_swap( T value ) {
  unsigned char *bytes = reinterpret_cast< unsigned char * >( &value );
  for (size_t i = 0; i < sizeof( T ); i += 2) {
    unsigned char v = bytes[ i ];
    bytes[ i ] = bytes[ i + 1 ];
    bytes[ i + 1 ] = v;
  }
  return value;
}

不用仔细阅读这个函数的实现。简单来说，这个函数传入一个 32 位的数字 0x11223344 会得到返回值 0x22114433。也可以传入一个 16 位的数字 0x1122 得到 0x2211。

但因为这是一个范型函数。自然可以传入一个 char。但这样就会访问到不属于自己的内存导致崩溃。

为了解决这个问题，我们自然可以引入模板特化。

1
2
3
4
5

template <>
char byte_swap( char value ) {
  assert( false && "Illegal to swap chars" );
  return value;
}

当 byte_swap 传入一个 char 类型的数据时，编译器将会为这次调用适配到这个特化的模板，此次调用将会触发断言。

注意看，这里的template后面的尖括号中，没有填写任何内容。这代表这是一个 模板全特化。全特化过的函数模板，由于在函数签名已经显著声明了类型 char value，因此不需要在尖括号里面写东西了。但如果没有形式参数，仍然需要额外添加内容，如以下代码中的 <int>：

1
2
3
4
5

template <class T>
void byte_not_swap(){ T foo; } // byte_not_swap 没有入参

template <>
void byte_not_swap<int>(){ int bar; }

除了全特化，我们还有一种特化方式叫 偏特化。这种特化方式不能用于函数模板。偏特化除了入参的类型，还需要其他的模板参数描述更多信息：

// A<char> 将会匹配到如下模板
template <class T1, class T2>
class A {
    ...
    private:
        T1 val;
        T2 val;
};

// A<int, char> 将会匹配到下面的偏特化模板
template <class T>
class A<int, class T2> {
    ...
    private:
        int val;
        T2 val;
};

让我们回到之前的 byte_swap 存在的问题，有了特化模板也许一定程度上解决了这个函数模板不能传入特定类型数据的问题。但显然我们需要为所有的非法数据类型都定义一份特化模板。这会产出多少重复代码？

能不能有一种方法，让我们的代码不需要写大量的特化模板就可以识别哪些是可以传入的参数，哪些是不能传入的参数呢？

有的，但显然只靠传进来的参数本身是不够的，我们需要能够识别传入数据的“类型本身的更多特征”来加强我们的模板，而这就是这就是 Type Traits 存在的意义。

Type Traits就是”类型的特征”的意思。在C++元编程中，程序员不少时候都需要了解一些类型的特征信息，并根据这些类型信息选择应有的操作。Type Traits有助于编写通用、可复用的代码。

那么接下来，我们开始旅程。

是惯例决定了元函数的定义——从和函数的对比来理解

元函数定义遵循以下标准

  • mf.1 元函数不是一个函数，是一个类或者结构体
  • mf.2 元函数并不是语言的一部分，也没有得到正式的语言支持
  • mf.3 元函数（相关编程方法）只是作为一种对现有语言特性的习惯用法存在的
  • mf.4 使用不受语言的约束
  • mf.5 由社区惯例约定（convention）约束用法

我们先把上面的定义记忆下来。

想要理解上面这些定义，和普通函数对比会更容易。然后我会把这些标准的体现用序号标注出来。

首先回顾一下什么是函数。

对于普通函数：

  • 可以传入零个或多个值。
  • 返回零个或一个值。

如下方所示：

1
2
3
4
5
6
7

// 声明
void do_something();
int do_something_else(int, char const*);
// 实现
int do_something_else(int, char const *) {
    return 42;
}

对于元函数，我们用粗体来标记差异：

  • 由于元函数只能是一个类或者结构体。所以元函数是一个拥有零个或多个模版参数的类/结构体。
  • 元函数可以返回零个或者多个返回值。
  • 但由于惯例，我们需要让元函数像普通函数一样只返回一个值。（但也有许多现存代码未遵守这一约定）

这确实让人不太容易想象类是怎么像函数一样工作的。这一切依然依赖惯例。

用一个最简单的元函数。来看看如何定义“类的返回值”：

通常有两类元函数，值元函数和类型元函数。

前者工作起来更像是一个普通函数，我们先只看这种：

依照惯例，值元函数返回值如何定义？

1
2
3
4

template <typename T>
struct TheAnswer {
    static constexpr int value = 42;  // 暴露一个public值，名字叫做 value
}

首先我们定义了一个结构体/类模板。自然，这种模板产生的只会是结构体或类（mf.1）。并不会真的出现一个什么“元函数”（mf.2）。而这个元函数的返回值，我们通过暴露一个名字叫做 value 的 public 值来实现一个“返回”值的效果（mf.3），马上就有例子来说明我们如何使用这个“返回值”。显然，这一切都是惯例决定的（mf.5）。即使你不使用value这个名字，编译器依然会让你编译通过（mf.4），但请不要这么做。

// 另一个普通函数
int int_identity(int x) {

// 对应功能的元函数
template <int X>         // 这是一个非类型模版参数，用模板的时候尖括号里面传具体值即可
struct IntIdentity {
    static constexpr value = X;  // 而这个具体值，在编译期就已经“存在” value 里面了
}

static_assert(42 == IntIdentity<42>::value); 

// 对应功能的范型元函数
template <typename T, T X> 
struct ValueIdentity{
    static constexpr T value = X;
}

static_assert(42ull == ValueIdentity<unsigned long long, 42ull>::value);  // 这里需要额外传一个类型

// 如果使用C++ 17 改写上面的代码...
template <auto X> 
struct ValueIdentity{
    static constexpr auto value = X;
}
 
static_assert(42ull == ValueIdentity<42ull>::value);                       // 类型可以省略掉了

可以看见，通过双冒号来访问这个public变量，我们得到了元函数的“返回值”。而为了实现范型，除了一个 X 值需要传给模板，还需要一个类型 T。

如果使用 C++ 17，则可以省略掉这个类型。如上方第三段代码所示。

我们使用了静态断言 static_assert 来表示结果的正确性。这是因为这些所谓的返回值，其实在编译期已经计算完成了。通过元编程，我们确实“把运行时的工作移动到了编译期”。而编译期的断言需要使用 static_assert，与运行期断言不同的是，如果你的结果计算错了导致断言为false，那么编译期就会报错。

再来一个另外一个例子，求两个数的和：

// 元函数
template <auto X, auto Y>
struct Sum {
    static constexpr auto value = X + Y;

static_assert(42ull == sum(30, 12ull));  // 传入了一个 int 值和一个 unsigned long long 值
                                         // 这个运算仍然在编译期就计算好了

                                        // 传入的值，也必须是一个编译期常量或常量表达式

但不得不说，constexpr 函数的存在确实让大多数情况下无需使用值元函数，但值元函数仍然有它的位置。

类型元函数返回值如何定义

接下来我们聊第二类元函数：类型元函数。这是模板元编程的主力。

类型元函数只能返回类型：

1
2
3
4
5

// PS：下面的实现与 C++ 20 引入的 std::type_identity 功能一致
template <typename T>
struct Echo {
    using type = T;                  // 暴露一个类型，名字叫做 type （必须！这是惯例）
}

和值元函数类似，我们暴露了一个名字叫做 type 的 T 类型。

类型元函数如何调用

如果编译器不知道一个东西是什么的时候，会优先认为那是一个值。因此我们不得不在调用类型元函数的前面，加上 typename 关键字来告诉编译器，这是一个类型：

1
2
3
4
5
6
7
8
9

// 调用值元函数
IntIdentity<42>::value

// 调用类型元函数
// 错误！编译器会认为type是一个值
TypeIdentity<int>::type

// 正确的写法
typename TypeIdentity<int>::type

但这种写法又很繁琐，因此惯例又规定了一种简化语法的方式：

使用变量模板 / 模板别名来简化元函数调用语法

如下：

// 使用变量模板，简化值元函数调用
template <auto X>
inline constexpr auto ValueIdentity_v = ValueIdentity<X>::value // 必须以 _v 结尾

static_assert(42ull == ValueIdentity_v<42ull>); 

// 使用模板别名，简化类型元函数调用 
template <typename T>
using TypeIdentity_t = typename TypeIdentity<T>::type;  // 必须以 _t 结尾

static_assert(std::is_same_v<int, TypeIdentity_t<int>>);  // 省了 typename！

这里使用了 is_same_v。其实不用说它是什么功能也能很轻松猜到了。首先 _v 是约定好的简写方法，用于简化::value，给它移除掉。可以猜测出这是一个is_same<typename X, typename Y> 的实例取返回值，返回值是bool。功能是对比两个类型是否是相同。
